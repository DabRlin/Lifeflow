## 已知问题

### 高优先级

1. **首页信息冗余**
   - 首页仍有问候语头部，与顶栏上下文信息存在部分重复
   - 建议：可考虑进一步精简或差异化展示内容

2. **习惯页面标题重复**
   - HabitsPage 仍保留 `<h1>习惯追踪</h1>` 标题
   - 与侧边栏、顶栏信息形成三重重复

### 中优先级

3. **热力图 N+1 查询问题**
   - HabitsPage 的打卡记录查询使用循环逐个请求每个习惯的记录
   - 影响：习惯数量多时性能下降
   - 建议：后端增加批量查询接口 `GET /api/checkins/batch`

4. **Header 组件数据获取过多**
   - 为显示上下文信息，Header 调用了 useTasks、useStatsOverview、useInfiniteLifeEntries
   - 建议：后端提供轻量级的 dashboard summary 接口

### 低优先级

5. **时间相关变量不自动更新**
   - `greeting` 问候语不会随时间变化自动更新
   - `today` 日期变量不会在午夜自动刷新
   - 影响：长时间不刷新页面时显示可能不准确

6. **Modal 组件样式不统一**
   - HabitsPage 中的 Modal 使用内联 style 而非 Tailwind
   - 建议：抽取为统一的 Modal 组件

## 技术债务

### 前端

1. **组件测试覆盖不足**
   - notification-system spec 中的 UI 组件单元测试未完成
   - 建议：补充 NotificationPanel、Header badge 等组件测试

2. **类型定义可优化**
   - 部分 API 响应类型使用 `any` 或不够精确
   - 建议：完善 TypeScript 类型定义

3. **错误边界缺失**
   - 缺少 React Error Boundary 处理运行时错误
   - 建议：添加全局错误边界组件

### 后端

1. **数据库迁移管理**
   - 当前使用 SQLAlchemy 自动创建表
   - 建议：引入 Alembic 进行正式的数据库迁移管理

2. **API 文档**
   - FastAPI 自动生成的文档可进一步完善
   - 建议：添加更详细的接口描述和示例
